Här förklarar jag följande termer

Commit

Commit är när man "sparar" sitt arbete, men man måste "pusha" det för att det ska komma in i servern

Push

Uppdaterar grenen med ens commits man gjort på grenen man arbetar på.

Pull

Om någon har ändrat på något i en seperat gren och vill att det ska bli kollat på och tillagt i huvudgrenen, så kan den personen lägga in en pull begäran. En pull begäran ber de ansvariga för repot att se över commitsen man har gjort, och sen merga ändringarna. En pull händer när man lägger till ändringar till huvudgrenen.

Merge

När man tar ändringar från en gren och lägger till det i en annan kallas det för en Merge, vanligtsvis huvudgrenen. Dessa commits läggs oftast först upp som en pull request innan de blir sammanslagna.

Lokalt vs Remote repo

Ett Remote repo är bättre när man ska arbeta i med andra. Sen kan man altid klona från servern och arbeta lokalt.

Working directory

Generells sätt, är den nyaste versionen "working directory", alltså där man jobbar just nu. varje fil man jobbar på, är här. Git upptäcker automatiskt ändringar.

Staging

Ett sätt att testa ändringar innan de läggs till i den faktiska GitHub Enterprise instansen.

Head

En speciell version av en gren, oftast den senaste commiten vid grenens spets.

Revert

När man Revertar en Pull request på GitHub, så öppnas automatiskt en ny pull begäran, och den har en commit som Revertar merge commiten från den orginella merge begäran. I Git kan man gå tillbaks med Git Revert.

Reset

Om man vill återställa det man jobbar på; alltså återställa till den senaste committen, så kan man använda en reset.

Repository

Ett Repo är en plats där man lagrar all data i sitt projekt, på datorn eller i molnet

Branch

En Branch är en förgrening i hierarkin, man kan grena ut från huvudgrenen och experimentera med olika ändringar där för att inte eventuellt förstöra hela projektet.

Clone

Man kan ladda ner ett repo från GitHub till sin hårddisk då Klonar man hela repot och skapar en kopia lokalt.

Markdown

Markdown är ett sätt att styla text på webben genom att definera vanlig text med några icke alfabetiska karaktärer. Markdown är vanligt i flera industrier och används ofta handel.

Forking

Forking är ett koncept medan cloning är en process. Forking är att bara ärhålla en seperat kopia av repository och det är ingen kommand inkluderat. Cloning är gjort genom kommandot git cloning och är en process genom att få alla kod filerna.
